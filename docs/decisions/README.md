# Architecture Decision Records (ADRs)

This directory contains the history of technical design decisions made for the AutoJMS project.

## What is an ADR?
An Architecture Decision Record (ADR) is a short document that captures a significant technical decision, its context, and its consequences.

## Format
Every decision is numbered sequentially and follows a template:
- **Title**: Short, descriptive name (e.g., `0001-setup-safe-vibe-coding.md`)
- **Status**: Proposed / Accepted / Rejected / Deprecated
- **Context**: What is the problem we are trying to solve? What forces are at play?
- **Decision**: What is the chosen solution?
- **Consequences**: What is the result of applying this decision (positive, negative, neutral)?
