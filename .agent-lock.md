# Agent Workspace Lock (GitHub Shared-Main Workflow Mode)

This lock governs the active developer agent write configuration on the local project directory.

## Status Lock
* **Current Writer**: Antigravity
* **Mode**: SHARED_MAIN_WRITE
* **Branch**: main
* **Scope**: none

---

## Allowed Operations
- Editing code files within active task scope.
- Restoring dependencies and compiling/building locally.
- Saving changes as commits on the local `main` branch.
- Pushing local commits to `origin/main` (using `ai-commit.ps1`) after all build/verify gates pass successfully.

---

## Forbidden Operations (Strictly Blocked)
- Force-pushing branches (`--force`).
- Rewriting Git history (rebasing public commits, amending pushed history).
- Executing hard resets (`git reset --hard`) after local commits have been pushed.
- Compiling, signing, or uploading production releases (`build-release.ps1 -Upload`).
- Bumping version numbers in `.csproj` files.
- Adding hardcoded credentials, secret keys, or service accounts.
