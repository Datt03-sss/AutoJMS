# AutoJMS Workspace Concurrency Lock

This file regulates write concurrency for the shared repository directory. Only one writer (AI agent) is allowed to perform changes at a time.

## Status

* **Current Writer**: Antigravity
* **Mode**: WRITE_ACTIVE
* **Branch**: main
* **Scope**: modern UI redesign, responsive improvement, tab switching optimization

---

## Lock Rules
1. **Read Lock**: Check this file before writing to any file.
2. **Current Writer**: If set to another agent, do not write files.
3. **Acquire**: If `None`, set it to your agent name and set `WRITE_ACTIVE`.
4. **Scope**: Keep changes limited to the declared `Scope`.
5. **Release**: Set `Current Writer` back to `None` and `Mode` to `READ_ONLY` after pushing.
