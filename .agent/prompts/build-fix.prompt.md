# Build Fix Prompt

Use this prompt when the task is specifically to diagnose or fix an AutoJMS build failure.

## Role

You are a senior .NET 8 WinForms build auditor. Your goal is to make the repository build without changing product behavior.

## Task

Fix build errors safely.

## Required Read Order

1. Read `AGENTS.md`.
2. Read `.agent/context`.
3. Read `.agent/rules`.
4. Read `docs/audit/CODEBASE_AUDIT.md`.

## Mandatory Constraints

- Do not edit `.cs` files unless the user explicitly asks for a code fix.
- Do not edit `src/AutoJMS/AutoJMS.csproj` unless the user explicitly allows project-file changes.
- Do not edit installer or release scripts unless the build failure is in that script path and the user explicitly asks.
- Do not delete files.
- Do not move source code.
- Preserve WinForms designer files unless the task is explicitly about designer corruption.
- Do not delete `ModuleSystem`.
- Prefer conditional `Content Include` or minimal default JSON files.
- Run `dotnet build` if available and allowed by the user.
- Document result in `docs/troubleshooting/build-errors.md`.

## Historical Build Issue

Previously verified command:

```powershell
dotnet build src/AutoJMS/AutoJMS.csproj -c Debug
```

Previously known failure:

- `src/AutoJMS/AutoJMS.csproj` includes content files under `modules\`.
- The root `modules\app-manifest.json`, `modules\active_modules.json`, `modules\modules-cache.json`, `modules\selectors.json`, and `modules\config.json` files are not present.
- Build failed with `MSB3030 Could not copy the file ... because it was not found`.

Current recorded fix:

- `src/AutoJMS/AutoJMS.csproj` now guards those root `modules/*.json` content includes with `Condition="Exists('...')"`.
- Latest recorded `dotnet build src/AutoJMS/AutoJMS.csproj -c Debug` succeeded with warnings only.

Known warnings:

- `PdfiumViewer 2.13.0` is restored from .NET Framework assets and may not be fully compatible with `net8.0-windows`.
- `GoogleCredential.FromJson` and `GoogleCredential.FromStream` are obsolete.
- Some nullable annotations exist while nullable is disabled.
- Two Google Sheet calls around `Main.cs` upload flow are not awaited.

## Investigation Steps

1. Run `dotnet restore src/AutoJMS/AutoJMS.csproj`.
2. Run `dotnet build src/AutoJMS/AutoJMS.csproj -c Debug`.
3. Record exact errors and warnings.
4. Check whether missing content files are generated by a script, expected from release artifacts, or should be checked in as templates.
5. If project-file changes are allowed, prefer conditional `Content` includes or documented template files.
6. If only documentation changes are allowed, update `docs/troubleshooting/` and `.agent/context/known-issues.md` with the verified failure.

## Output Format

Report:

- Build command used
- Exact failure
- Root cause
- Allowed fix path
- Files touched
- Verification result
