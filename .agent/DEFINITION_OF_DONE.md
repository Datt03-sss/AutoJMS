# Definition of Done

A task is done only when:

## For documentation tasks

- [ ] Files created/updated
- [ ] No production code modified
- [ ] Summary printed
- [ ] Next recommended task stated

## For build fixes

- [ ] Root cause identified
- [ ] Minimal patch applied
- [ ] Build command documented
- [ ] Build result recorded
- [ ] No runtime logic changed

## For bugfixes

- [ ] Context read
- [ ] Root cause stated
- [ ] Files changed listed
- [ ] Patch is minimal
- [ ] No unrelated refactor
- [ ] Acceptance criteria verified
- [ ] Rollback note provided

## For release tasks

- [ ] VelopackVersion is valid SemVer
- [ ] GitHub Release assets verified
- [ ] Supabase manifest files verified
- [ ] No `.nupkg` uploaded to Supabase
- [ ] App does not open GitHub page
- [ ] Manual update via tab About remains correct

## For tier tasks

- [ ] BASE manual features still work
- [ ] BASE background jobs disabled
- [ ] ULTRA behavior preserved
- [ ] FullStack remains separate form
